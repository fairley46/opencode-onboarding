# Module 8: Asking Questions Across Tools

## Why This Matters

One of the fastest ways to get real value from OpenCode is to ask a question that uses more than one approved source. This is where beginners often start to see why an agent workflow can be useful.

## The Simple Picture

You can ask one question that combines information from multiple approved tools or sources. The trick is to be explicit about what sources should be used and what kind of answer you want back.

## A Practical Example

Instead of saying, "Tell me everything about this," try:

"Use the approved docs source and the policy reference source. Give me a short answer, list the key differences, and tell me what I should confirm with a human."

That prompt gives the system:

- the sources to use
- the output format
- the review boundary

## Why Beginners Struggle Here

Broad questions create broad answers. Narrow questions create more trustworthy answers.

## Safety Check

When the answer matters, ask for source attribution. That keeps the workflow easier to review and easier to correct.

## What To Remember

If you want a better answer, name the sources, the goal, and the output shape.
